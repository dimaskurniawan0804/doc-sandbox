## 1. High-Level Summary (Apa yang Berubah?)

Perubahan ini memigrasikan sumber data lokasi (negara, provinsi, kota, kecamatan, kelurahan) pada alur KYC V2 dari **Legacy Regional API V1** ke **Regional API V2** yang baru. ### 2.2 Regional Store (Modul Baru)

#### [NEW] [Regional.store.ts](file:///Users/skilab047/Documents/SKI/web/src/store/pinia/master/v2/module/regional/Regional.store.ts)

  - `fetchDivision3(id)` → `GET /v2/master/regions/division3/{id}` (Kecamatan)
  - `fetchDivision4(id)` → `GET /v2/master/regions/division4/{id}` (Kelurahan)

#### [NEW] [regional.respond.dto.ts](file:///Users/skilab047/Documents/SKI/web/src/store/pinia/master/v2/module/regional/dto/regional.respond.dto.ts)

  - `Division1ResponseDTO` — `isoCode`, `name`, `localName`, `groupCode`, `governmentCode`
  - `OtherDivisionResponseDTO` — `id`, `name`, `localName`, `governmentCode` (dipakai untuk Division 2, 3, 4)

- **💡 SUMMARY HASIL PERUBAHAN (ATURAN/PERILAKU BARU SISTEM)**:
  `RegionalStore` adalah **data-access layer murni**; ia tidak menyimpan state. Fungsi-fungsinya hanya melakukan HTTP GET dan mengembalikan data mentah. Semua transformasi (mapping ke `KycV2SelectOptionType`, sorting, penambahan opsi "Other") tetap tanggung jawab `KycV2.store.ts`.

### 2.4 KycV2 Store — Fungsi Province (`getProvinceV1` + `getDivision1V2`)

#### [MODIFY] [KycV2.store.ts](file:///Users/skilab047/Documents/SKI/web/src/store/pinia/kycV2/KycV2.store.ts#L1608-L1671)

- **BEFORE**: `getProvinceV1(countryNumericCode, locale)` memanggil `/master/countries/{numericCode}` dan memetakan `divisions` dari response. Label ditentukan oleh parameter `locale`.
- **AFTER**:
  - Jika `usesRegionalAPIV2`, fungsi mencari `iso2Code` dari `countryOptions` (shortcut `"ID"` untuk `numericCode === "360"`), lalu memanggil `getDivision1V2(iso2Code)`.
  - `getDivision1V2(iso2Code)` tidak lagi menerima parameter `locale`. Label ditentukan secara internal: `iso2Code.toUpperCase() === "ID"` → gunakan `localName`, selainnya → gunakan `name`.

  - Pencarian `numericCode` menggunakan `String()` comparison: `String(c.options?.numericCode) === String(countryNumericCode)` untuk menghindari type mismatch (number vs string).

- **💡 SUMMARY HASIL PERUBAHAN (ATURAN/PERILAKU BARU SISTEM)**:
  1. **Label = Value** pada V2 path. Ini adalah perubahan perilaku penting: pada V1, `value` bisa berbeda dari `label` (misal: `label: "Jawa Barat"`, `value: "West Java"`). Pada V2, keduanya identik.
  2. **Penentuan bahasa bukan dari `locale` parameter**, melainkan dari `iso2Code` (untuk Division 1) atau `selectedCountryNumericCode` (untuk Division 2–4).
  3. `selectedCountryNumericCode` disimpan saat `getProvinceV1` sukses dan digunakan oleh level di bawahnya.

### 3.2 Alur OCR Auto-Fill Address (Sekuensial)

```
onMounted (KycV2DataVerification.vue)
  │
  ├─ OCR polling selesai → data province dimuat via getProvinceV1("360", "id")
  │
  ├─ province = userDataForVerification.addressInformation.province()
  │   └─ sinkron: find() di provinceList → return match atau placeholder
  │
  ├─ city = await userDataForVerification.addressInformation.city()
  │   └─ await mapCityListV1(province.label) → lazy fetch division2
  │   └─ find() di cityList → return match atau placeholder
  │   └─ GUARD: if (!isComponentMounted) return
  │
  ├─ subDistrict = await userDataForVerification.addressInformation.subdistrict()
  │   └─ find city di cityList → await mapSubdistrict(city.label) → lazy fetch division3
  │   └─ find() di subdistrictList → return match atau placeholder
  │   └─ GUARD: if (!isComponentMounted) return
  │
  ├─ ward = await userDataForVerification.addressInformation.ward()
  │   └─ find subdistrict di subdistrictList → await mapWard(subdistrict.label) → lazy fetch division4
  │   └─ find() di wardList → return match atau placeholder
  │   └─ GUARD: if (!isComponentMounted) return
  │
  └─ catch (error):
      └─ Reset city, subDistrict, ward ke placeholder
      └─ User memilih manual via dropdown
```

### 3.3 Lookup Key per Level (V2)

| Level | List Source | Lookup Key untuk Level Berikutnya | Parameter Fetch |
|---|---|---|---|
| Country | `countryOptions` | `options.iso2Code` + `options.numericCode` | — |
| Province (Div 1) | `provinceList` | `options.iso2Code` (ISO code provinsi) | Country `iso2Code` |
